Desarrollo de un módulo (en Java 8) de elaboración de pruebas para...

Publicado: Hace 29 días Plazo: 30 días Propuestas: 16
Título: Desarrollo de un módulo (en Java 8) de elaboración de pruebas para imprimir y tomar a los estudiantes
Tema: Desarrollo de un módulo de elaboración de evaluaciones (Pruebas con preguntas) para estudiantes en JAVA 8, para un software de escritorio.

Competencias del postulante:
-    Desarrollador en java 8 escritorio- netbeans
-    Conocimiento en jasper-report  5.5
-    Gestor de base de datos Postgres 9.5
-    Conocimiento en librerías aspose para gestionar archivos en java. Oh una alternativa mejor.

Generalidades.
Buscamos la realización de un módulo el cual se va a integrar a un software (Software de escritorio desarrollado en JAVA). Este módulo es de elaboración de pruebas, consiste en la posibilidad de realizar evaluaciones (Solo realizar evaluaciones para imprimir y tomar a los estudiantes de manera presencial).

Las características del módulo son:
-    Las evaluaciones van a estar segmentadas por curso, materia y por parcial (Parcial hace referencia a un rango de inicio y finalización de un periodo, tenemos 8 periodos 6 parciales y 2 quimestrales; cada 3 parciales hay 1 evaluación quimestral).
-    Tener la posibilidad de realizar evaluaciones desde 0.
-    Tener un archivo de evaluaciones guardadas que se puede editar para personalizar a lo que el docente requiera, esto tanto para evaluaciones parciales, quimestrales y anuales.
-    La evaluación debe permitirte aparte de ser guardadas en el software o imprimirlas, tener la posibilidad de guardar en una carpeta externa de Word o excel.
-    Previo a cada pregunta, en un espacio debe permitir escoger una destreza e indicador que van a estar relacionados al módulo de planificaciones en donde se encuentra esto. Además de que pueda, para escoger la destreza e indicador, filtrar y que me salgan menos. 
-    Debe permitir realizar diferentes tipos de preguntas de BASE (Adjunto el modelo de la diversidad de formas de preguntas).




-    Adjuntar imágenes tanto para las preguntas como para las opciones de respuesta.
-    De una misma evaluación tener la posibilidad de redistribuir aleatoriamente las preguntas y de cada pregunta las alternativas de respuesta a fin de tener varias alternativas (Las que yo escoja 2,  3, 4 o 5) para imprimir que haya menos posibilidad de copia.
-    La evaluación quimestral debe tener la posibilidad de crear preguntas o escoger entre las preguntas que he ubicado en las evaluaciones de los parciales (Al final de 3 parciales hay 1 evaluación quimestral)
-    Al final de año debe permitir crear evaluaciones de supletorio, remedial y de gracia en las que tenga la posibilidad de crear preguntas pero también escoger de entre las preguntas de las evaluaciones tanto de parcial como de quimestre.  (Estas evaluaciones se toman cuando un estudiante no ha logrado el mínimo en sus notas y debe entrar en un proceso de suspensión)
-    Cada evaluación debe tener su hoja de respuestas para el docente y que esto le permita calificar rápidamente.
-    Tener la posibilidad de tener una prueba con las respuestas marcada.
-    Asignar los puntos que va a tener cada pregunta y con ello me vaya calculando.
-    Las preguntas (Con sus opciones) deben permitirse arrastrar de lugar, es decir que si la pregunta que he puesto 3 la quiero primero, lo pueda hacer.
